## 1. 基础约定

二轮充电是独立模块，接口、订单、支付、站桩枪、白名单、月卡、充值卡均使用二轮专用接口和 `two_wheel_*` 表，不复用四轮汽车充电的 `charging_*` 表和 `/pay/charging` 接口。

`BKV` 只是设备品牌值，写入 `device_brand='BKV'`。后续接入其他二轮设备品牌时，业务接口仍保持 `/twoWheel/*`，只在设备协议适配层按 `device_brand` 分发。

通用响应：

```json
{
  "success": true,
  "data": {}
}
```

失败响应：

```json
{
  "success": false,
  "error": "错误原因"
}
```

通用分页参数：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| page | number | 否 | 页码，默认 1 |
| page_size | number | 否 | 每页条数，默认 20，最大 200 |

## 2. 数据库表说明

所有表均包含 `id`、`created_at`、`updated_at`、`deleted_at`、`deleted`。业务编号字段例如 `station_no`、`pile_no`、`port_no`、`order_no` 不替代主键 `id`。

### 2.1 two_wheel_station

| 字段 | 说明 |
| --- | --- |
| id | 站点主键 |
| station_no | 二轮站点编号 |
| two_wheel_station_no | 外部或兼容编号 |
| name | 站点名称 |
| data_owner_id | 数据归属 |
| parking_lot_id | 关联停车场 |
| address/province/city/district | 地址信息 |
| longitude/latitude | 坐标 |
| status | enable/disable |
| device_brand | 默认 BKV；站点可包含多个品牌桩，桩表字段为准 |
| two_wheel_strategy_ids | 站点可用计费策略配置 |
| whitelist_apply | 白名单是否参与免支付判断 |
| invoice_apply/invoice_info | 发票配置 |
| options/remark/comment | 扩展和备注 |

### 2.2 two_wheel_pile

| 字段 | 说明 |
| --- | --- |
| id | 桩主键 |
| pile_no | 桩编号 |
| device_no | 设备编号 |
| two_wheel_station_id | 所属站点 |

| brand | 品牌展示名 |
| type/device_type | 设备类型 |
| pile_status | enable/disable |
| using_status | stock/online/offline/fault 等 |

| qrcode_id/operator_id | 二维码、运营方 |
| last_online_at | 最近在线时间 |
| options/remark | 扩展和备注 |

### 2.3 two_wheel_port

## 4. 后台管理接口

### POST /twoWheel/station/create

创建二轮站点。

请求：

```json
{
  "station_no": "TWS001",
  "name": "东门二轮充电站",
  "data_owner_id": "DATA_OWNER_ID",
  "parking_lot_id": "PARKING_LOT_ID",
  "address": "园区东门",
  "longitude": 119.30000000,
  "latitude": 26.08000000,
  "status": "enable",
  "whitelist_apply": true,
  "two_wheel_strategy_ids": { "default": "STRATEGY_ID" },
  "device_brand": "BKV"
}
```

响应 `data` 返回 `two_wheel_station` 完整记录。

### POST /twoWheel/station/update

更新站点任意业务字段，不只是改名。

请求：

```json
{
  "id": "STATION_ID",
  "name": "东门二轮充电站A区",
  "status": "enable",
  "whitelist_apply": false,
  "service_phone": "4000000000"
}
```

响应 `data` 返回更新后的站点记录。

### POST /twoWheel/station/delete

软删除站点。

请求：

```json
{ "id": "STATION_ID" }
```

响应：

```json
{ "success": true, "data": { "id": "STATION_ID" } }
```

### POST /twoWheel/station/detail

## 7. 支付和权益规则

`/pay/twoWheelCharging` 是充电订单的唯一二轮支付入口。普通微信、支付宝等线上支付传 `identity_source=user`；月卡传 `monthly_card`；充值卡传 `charging_card`；白名单传 `whitelist`。

`POST /twoWheel/xcx/pay/order`、`POST /twoWheel/xcx/pay/monthCard`、`POST /twoWheel/xcx/pay/rechargeCard` 不再作为 v2 接口使用。

二轮支付会生成 `two_wheel_pay_order`，同时接入平台 `OutTrade/InternalTrade`。三类二轮内部流水类型为：

| trade_type | 说明 |
| --- | --- |
| two_wheel_charging | 普通二轮充电订单支付 |
| pay_two_wheel_month_card | 二轮月卡购买或续费 |
| pay_two_wheel_charging_card | 二轮充值卡充值 |

月卡和充值卡作为充电权益时，不再额外创建线上支付流水；充电结束时按订单实际时长或金额扣减，并写对应流水。
